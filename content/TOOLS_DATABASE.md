## Tool Selection Criteria (For Client Consultations)

When recommending tools to a client, evaluate each option against:

1. **Cost vs. Budget** — Is it affordable for this client's size? Free tier available?
2. **Internet Dependency** — Will it work reliably with Tanzanian internet speeds?
3. **Learning Curve** — Can staff learn it in 1-2 training sessions?
4. **Mobile Accessibility** — Does it work on smartphones? (Many TZ businesses are mobile-first)
5. **Swahili Support** — Does it handle Swahili input/output well?
6. **Integration** — Does it connect with tools they already use?
7. **Data Privacy** — Where is data stored? Any regulatory concerns?
8. **Scalability** — Will it grow with the business?

## Notes on Tanzania-Specific Considerations

- **Internet reliability:** Favor tools that work offline or with poor connectivity. Avoid tools that need constant high-bandwidth connections.
- **Mobile-first:** Many staff will access tools via phone. Prioritize mobile-friendly tools.
- **WhatsApp dominance:** Any customer-facing AI should have a WhatsApp integration path.
- **Cost sensitivity:** Free tiers and per-user pricing are more attractive than flat enterprise fees for most TZ businesses.
- **Data sovereignty:** Some industries (banking) may have requirements about where data is processed. Flag this during discovery.
- **Payment methods:** Some global SaaS tools don't accept Tanzanian payment methods easily. Consider this in recommendations.
